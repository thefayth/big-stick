# Roadmap

This roadmap is public-safe and non-binding. It does not promise launch dates,
availability, pricing, licensing, or access.

## Phase 0: Protect And Document

- Maintain the public/private boundary.
- Keep source, records, prompts, workflows, and receipts private.
- Prepare public-safe docs and visual briefs.

## Phase 1: Private Prototype / Internal Review

- Continue local app verification.
- Review dashboard, public record, vault, Listen Up, AI Console, exports, and
  settings flows privately.
- Keep AI drafts review-first.

## Phase 2: Public Project Page

- Publish a public-safe FaithCheltenham.com page only after Faith review.
- Use abstract visuals rather than screenshots of private records.

## Phase 3: Private Pilot Or Partnership Review, If Faith Chooses

- Discuss private demo or pilot boundaries.
- Define data, privacy, evidence, and review rules before any access.

## Phase 4: Future Release Or Licensing, If Faith Chooses

- Consider licensing, services, or controlled release only with explicit written
  policy, terms, and safety review.
