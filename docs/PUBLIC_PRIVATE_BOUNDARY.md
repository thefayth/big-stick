# Public / Private Boundary

## Public

- High-level description of Big Stick.
- Public-safe project story.
- Public-safe abstract visuals and diagrams.
- Public-safe status.
- Non-sensitive roadmap.
- Public inquiry path.
- WordPress-ready draft copy after Faith review.

## Private

- Source code unless Faith explicitly approves a public release.
- Agent instructions.
- Private prompts.
- Credentials.
- Adapters.
- Workflows.
- Private receipts.
- Customer data.
- Admin, legal, medical, benefits, and family documents.
- Raw evidence and transcripts.
- Private screenshots.
- Strategy memos.
- Unpublished art, manuscripts, or narrative material unless explicitly chosen.
- Production systems.
- Deployment details.
- Financial details.
- Internal notes.

When in doubt, keep it private.
